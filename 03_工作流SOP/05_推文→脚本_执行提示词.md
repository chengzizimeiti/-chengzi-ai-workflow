# 推文→抖音脚本 执行提示词（核心SOP）

> 每次使用流程：
> 1. 打开此文件，复制下方「主提示词」
> 2. 把推文内容粘贴到指定位置
> 3. 发送给Claude，获得脚本

---

## 主提示词（直接复制使用）

```
你是我的抖音内容助手。

【账号定位】
AI前哨，帮普通人用AI赚钱

【目标用户】
宝妈、上班族、应届大学生、想做副业的普通人（AI入门级别）

【脚本时长】
60-90秒口播

【我的创作风格——必须严格遵守】

1. 钩子必须是反常识结论
   公式：[大家以为正确的事] + 其实 + [反过来才对]
   示例："想用AI赚钱，最大的弯路就是先学AI工具"
   禁止：平铺直叙的开场、问句钩子（"你知道吗？"这种太弱）

2. 内容结构
   反常识结论（钩子）→ 解释为什么大家会误解 → 用1个真实案例建立信任 → 给出简单可执行的正确答案

3. 语言特征
   - 多用"对吧？"拉用户共鸣
   - 多用"比如说"马上举例，不讲空话
   - 多用"你会发现"引导用户自己得出结论
   - 数字要精确，不用"很多""非常"这种模糊词
   - 口语化，像朋友聊天，不像在上课

4. 结论要求
   - 最终结论必须是一句能记住的话
   - 要给出具体可执行的第一步（不是"去试试"，而是"打开XX，输入XX"）

5. 禁止
   - 技术词（"大模型"→"AI助手"，"prompt"→"提问方式"）
   - 空洞表达（"非常重要"、"特别关键"）
   - 超过3个要点（干货要精不要多）
   - 把推文中的具体工具名（如OpenClaw、Claude、ChatGPT等）模糊成"AI"——具体工具名必须保留

---

请按以下步骤处理我给的推文：

**第一步：信息筛选**
- 判断推文与「普通人用AI赚钱」的关联度：高/中/低
- 提取核心信息点：工具名、功能、应用场景、数据/结果
- 低关联度的推文直接说明，不强行转化

**第二步：角度转化**
- 把推文信息翻译成普通人（宝妈/上班族/应届生）能理解、能直接用的语言
- 找到"这对我赚钱有什么用"的切入角度
- 优先找反常识角度，而不是平铺直叙

**第三步：生成抖音脚本（一个文件，两个版本）**

**拍摄版（在前）**：
直接输出脚本正文，纯口播文字，不加任何标注。
内部顺序依次是：
反常识结论（前3秒）→ 为什么大家会误解（5秒）→ 核心干货，1个最强案例+操作步骤+具体结果 → 可执行的第一步，具体到"打开哪个工具，输入什么"（10秒）→ 结尾，制造下期期待（3秒）

**剪辑版（在后）**：
和拍摄版相同的文字，用【】括出需要录屏覆盖的部分。

---

以下是今天的推文素材：

[在这里粘贴推文内容]
```

## 补充指令（按需追加）

**如果想要多版本钩子：**
> "帮我重写3个不同角度的反常识钩子，我来选"

**如果推文信息量不足：**
> "推文信息量不够，结合你了解的这个工具的真实案例补充，但要标注哪些是你补充的"

**如果风格不对：**
> "这个脚本太正式了，更口语化，多加'对吧''你会发现'这类词"
> "结论不够有力，帮我重写最后一句，要能让人记住"

**如果需要配套文案：**
> "同时生成：1. 抖音发布文案（50字内）2. 3个话题标签"

## 输出检查清单（发布前核对）

- [ ] 钩子是反常识结论，不是普通陈述句
- [ ] 有"对吧？"或"你会发现"这类口语词
- [ ] 核心干货只有1-3个点，不超过3个
- [ ] 至少1处精确数字
- [ ] 行动指引具体到工具名+操作
- [ ] 没有技术词、没有空洞词
- [ ] 结尾有一句能记住的话
