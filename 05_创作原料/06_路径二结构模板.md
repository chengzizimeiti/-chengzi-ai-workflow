# 路径二结构模板（AI小知识系列）

> 每天讲清一个AI小知识，科普/介绍类，不是路径一的反常识打法。
> 选题必须追热度——只做当下AI圈有热度的话题，目的是建立橙子"AI专家"人设。
> 选题由显眼包搜索热点后提供，橙子选定再出脚本。详见 `02_选题库/选题边界.md`。

---

## 固定结构（三段式）

### 一、开头（固定句式）

```
每天榨干一个AI知识，[知识点名称]是什么？
```

> 这句话固定不变，只替换知识点名称。
> ⚠️ **开头的知识点名称必须和正文内容严格对应**。如果正文只讲了某工具的某个子功能，开头就必须写那个子功能名，不能用更宽泛的工具名代替。例：正文只讲即梦的Seedance2.0视频生成，开头就写"Seedance2.0是什么"，不能写"即梦是什么"。

### 二、正文（大白话讲清楚）

- 用最通俗易懂的大白话解释这个知识点
- 必须举例子，让人一听就懂
- 例子要贴近目标用户的日常（宝妈、上班族、大学生能想象到的场景）
- **举例必须从目标用户的真实日常场景出发**（宝妈带娃/做副业、上班族日常工作、大学生找实习找工作），**禁止用创业者/老板/HR等非目标用户的视角举例**
- 不用技术词，不用反常识，不用泼冷水，就是**讲明白**
- **解释功能时，必须给出至少2个具体场景**，不能只说一个，让不同用户都能对号入座
- ⚠️ **例子的获得感必须落到钱**：省钱/赚钱/占便宜。基于橙子账号定位"帮普通人用AI赚钱"，例子不能以情感价值、社交价值、成就感等软性回报收尾，必须让用户感受到真实的金钱收益或节省
- ⚠️ **例子要能撑起录屏节奏**：举例时考虑剪辑版录屏的过程感，不能只是"输入一句话→等结果"，要有多个步骤、对比、或有分量的结果画面

### 三、收尾（说明有什么用）

- 这个小知识对普通人有什么实际用处
- 落点：能帮你赚钱、省时间、或者避坑

### 四、结尾钩子（固定slogan）

```
关注橙子，每天榨一点AI
```

> 每条路径二脚本结尾固定加这句，不做修改。

## 语气要求

- 像朋友聊天，不像上课
- 平和、亲切、好懂
- **不用**路径一的狠话（"屁用没有""大坑"）
- **不用**反常识开头
- **不用**情绪词收尾

## 关键词标记规则

- 脚本中需要重读、停顿、或后期重点处理的词用「」标记
- 目的：方便拍摄时调整语气变化，以及后期剪辑定位
- 每句话不超过1-2个标记，不要全篇都标

## 脚本输出格式（一个文件，两个版本）

每条路径二脚本在同一个md文件里输出两个版本：

### 拍摄版（在前）
- 纯口播文字，橙子拿来对着镜头念就行
- 不带任何拍摄/剪辑标注

### 剪辑版（在后）
- 和拍摄版相同的文字内容
- 用【】括出需要录屏覆盖的部分，橙子剪辑时把这些地方的口播画面替换成对应的录屏素材

## 示例骨架

```
每天分享一个AI知识，提示词是什么？

你用ChatGPT或者Claude的时候，打字问它问题，你打的那句话就叫提示词。
比如说你跟AI说"帮我写一条朋友圈文案"，这句话就是提示词。
但如果你说"我是卖水果的，帮我写一条朋友圈文案，要接地气，带上今天榴莲到货了"，
这也是提示词，但效果比前面那句好十倍。
简单说，提示词就是你跟AI说话的方式，你说得越清楚，AI给你的东西越好用。

知道这个有什么用呢？
你以后用AI帮你干活，不管是写文案还是做表格，
把你的需求说具体——你是干什么的、要给谁看、要什么风格，
出来的东西直接就能用，不用反复改，省的就是真金白银的时间。
```

## 路径二 vs 路径一

| | 路径一（推文转口播） | 路径二（AI小知识） |
|--|---|---|
| 开头 | 反常识钩子 | "每天分享一个AI知识，XX是什么？" |
| 正文 | 泼冷水→揭真相→案例 | 大白话解释+举例子 |
| 收尾 | 情绪词收尾 | 说明这个知识对你有什么用 |
| 语气 | 重、敢说 | 平和、亲切、好懂 |
| 目的 | 制造认知冲击 | 讲明白一个东西 |
