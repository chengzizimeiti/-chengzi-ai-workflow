# 橙子·创作风格分析文档

> 基于真实文案提炼，用于指导脚本生成方向

---

## 一、钩子风格

**核心特征：反常识开场**

几乎每条内容都用一个"听起来不对劲"的结论开头，逼着用户停下来想"为什么"。

| 真实案例 | 反常识结构 |
|---------|-----------|
| "一个人赚不到钱的核心原因就是你上班上多了" | 上班 → 赚不到钱（反直觉） |
| "世界上离钱最远的地方就是我想基于自己的擅长去赚钱" | 擅长 → 离钱最远（反直觉） |
| "赚钱最大的弯路叫做我想找一个自己擅长的事情去赚钱" | 同上 |
| "99%的人不需要自动化" | 自动化 → 不需要（反直觉） |

**钩子公式**：`[大家以为正确的事] + 其实是 + [反过来才对]`

---

## 二、内容结构

**标准叙事路径**：

```
反常识结论（钩子）
    ↓
解释为什么大家会误解（同理心）
    ↓
拆框架 / 举真实案例（建立信任）
    ↓
数字/排列组合放大认知冲击（让人"哇"）
    ↓
给出简单可执行的正确答案（价值感）
    ↓
用自身案例作为终极背书
```

**标志性手法**：
- **排列组合放大法**：把一个问题拆成N个维度，7×7=49，49×2=98……让用户意识到问题比想象中复杂，然后给出简单解法
- **案例对比法**：用自己的真实经历（投流卖首饰、PDF分销、AI社群）做有血有肉的证明
- **子话题打假**：先认真回答一个"错误的问题"，然后说"这其实是个错误的问题"

## 三、语言特征

**高频词汇/句式**：
- "对吧？" —— 拉用户入戏，确认共鸣
- "比如说" —— 马上举例，不讲空话
- "你会发现" —— 引导用户自己得出结论
- "所以" —— 频繁转折推进逻辑
- "就是" —— 口语化的确认语气
- "我管这个叫做XXX" —— 给概念命名，增强记忆点
- "焦虑内耗" —— 目标用户的核心情绪状态，高频使用
- "屁用没有"、"大坑"、"从来不是" —— 敢说重话，不温吞

**数字使用习惯**：
- 精确引用自身数据（"三十多万阅读量"、"两万多点赞"、"一万三千条帖子"）
- 用数字放大问题规模（"六百八十六种情况"）
- 不用模糊词，用具体数字

**禁止出现的表达**：
- 模糊词："可能"、"也许"、"或许"（除非故意反用）
- 空洞词："非常重要"、"特别关键"
- 学术词：保持口语
- 喊口号式行动指令："现在就做一件事……"——用情绪词收尾更有力

## 四、内容价值观（影响选题角度）

| 价值观 | 具体体现 |
|--------|---------|
| **反内耗** | 批判过度准备、分析瘫痪、"模拟创业" |
| **反赛道思维** | 不看行业，看具体生意 |
| **执行优先** | "把事儿干了"是核心结论 |
| **系统思维** | 把生意/行为拆成可复制的系统 |
| **真实数据背书** | 用自己的真实结果说话，不造假 |

**目标用户的真实状态（必须准确描述）**：
- 连ChatGPT/Claude都没用过，不知道什么是提示词
- 被大V内容刷到，核心恐惧是"被AI时代淘汰"
- 处于焦虑内耗状态，想行动但不知道从哪开始
- 离那些复杂工具/系统"非常遥远"

## 五、Douyin脚本适配规则

长文章 → 60-90秒口播的转化原则：

| 长文元素 | Douyin适配方式 |
|---------|--------------|
| 反常识钩子 | 直接用，前3秒，后面立刻跟具体案例/工具名 |
| 排列组合放大法 | 压缩为"有X种情况，但你只需要知道1个" |
| 多个案例 | 只保留1个最强的 |
| 复杂框架 | 提炼为3个字的口诀或1句话结论 |
| 自身数据背书 | 保留，放在中间建立信任 |
| 最终结论 | 用情绪词收尾（焦虑/内耗/遥远），不用喊口号式行动指令 |
| 用户画像 | 在脚本中途主动说出用户的真实处境，建立强共鸣 |

**画面感写法**（参考真实改动）：
- 不写流程："几个Agent来回开会讨论"
- 写感受："你会感觉你的算力在燃烧"

---

## 六、一句话风格定义

> 用反常识的结论打开认知，用真实的案例建立信任，用最简单的结论给出行动指令。
