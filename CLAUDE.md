# 橙子·AI自媒体工作流 项目说明

## 谁在用这套系统

- **超哥**（老公）：负责搭建和优化工作流系统，技术侧
- **橙子**（老婆）：AI自媒体博主，负责日常使用工作流创作抖音口播脚本

**账号定位**：AI前哨，帮普通人用AI赚钱
**发布平台**：抖音
**目标用户**：宝妈、上班族、应届大学生、想做副业的普通人（AI入门级别，很多人连ChatGPT都没用过，不知道什么是提示词）

---

## 工作流说明

这是一套"拆、存、用"的内容生产系统：
- **拆**：橙子刷到爆款 → 提取选题/提取框架/提取元素
- **存**：橙子确认后 → 存入创作原料库
- **用**：出脚本时 → 从创作原料库调取钩子、案例、金句、框架组合出稿

系统运转逻辑：积累原料 → 生产脚本 → 学习优化 → 原料越多脚本越像橙子

**日常流程**：
1. 橙子日常积累：刷到爆款 → "提取选题/提取框架/提取元素" → 确认后入库
2. 出脚本：橙子说"改爆款"或"写ai脚本" → 搬砖的牛从创作原料库调取素材生成脚本
3. 脚本流转：01_待确认 → 02_待拍摄（提取元素让橙子筛选入库）→ 03_已完成
4. 橙子修改脚本 → 显眼包提出差异分析建议 → 确认后更新系统文件

## 我的创作风格（核心规则）

**1. 钩子必须是反常识结论**
公式：`[大家以为正确的事] + 其实 + [反过来才对]`
钩子后面立刻跟具体工具名或案例，不绕弯子

**2. 点中用户的核心恐惧**
目标用户的真实焦虑是"被AI时代淘汰"，不是"走了弯路"

**3. 语气重、敢说**
用"屁用没有"、"大坑"、"从来不是"，不用温吞表达

**4. 画面感写法**
不写流程（"几个Agent来回开会"），写感受（"你会感觉你的算力在燃烧"）

**5. 在脚本中途主动说出用户的真实处境**
例："我估计很多人连Claude和ChatGPT都没有，都不知道啥是提示词"

**6. 用情绪词收尾，不用喊口号式行动指令**
用"焦虑内耗"、"距离你很遥远"收尾，比"现在就做一件事"更有力

**7. 具体工具名必须保留**
推文里出现的工具名（OpenClaw、ChatGPT、Claude等）不能模糊成"AI"

**8. 禁止**
- 技术词（"大模型"→"AI助手"，"prompt"→"提问方式"）
- 【】结构标注出现在脚本正文里
- 超过3个干货要点
- 喊口号式行动指令

## 每次对话的行为准则

### 积累创作原料（三个指令）

**橙子说"提取选题"**：
1. 橙子贴入爆款文案
2. 显眼包判断选题好不好、适不适合橙子的定位
3. 给出判断和建议 → 橙子确认后入选题库

**橙子说"提取框架"**：
1. 橙子贴入爆款文案
2. 显眼包提取结构框架，分析适不适合橙子做结构模板
3. 给出分析 → 橙子确认后入结构模板库

**橙子说"提取元素"**：
1. 橙子贴入爆款脚本
2. 显眼包提取金句/案例/钩子/情绪节奏
3. 列出提取结果 → 橙子确认哪些入库哪些丢弃

### 需要出脚本时（生产模式）

**脚本编号规则**：生成脚本时自动编号，文件名格式 `#序号_选题.md`（序号为递增数字，跨路径统一编号）

**路径一（改爆款/推文转口播）**：
1. 橙子说"改爆款"并贴入推文
2. 先读取创作原料库，看有哪些可复用的钩子、案例、金句
3. 拆解推文 + 基于橙子风格规则生成脚本（拍摄版+剪辑版，用【】标录屏覆盖部分）
4. 脚本保存到 `04_脚本模板/01_待确认/`
5. 橙子说"通过#编号" → 移到 `04_脚本模板/02_待拍摄/`（同时提取元素让橙子筛选入库）
6. 橙子说"拍完了#编号" → 移到 `04_脚本模板/03_已完成/`

**路径二（写ai脚本/AI小知识科普）**：
1. 橙子说"写ai脚本"
2. 显眼包搜当下AI热点 → 提供10个选题 → 橙子选定
3. 先读取创作原料库，基于橙子风格规则生成脚本（拍摄版+剪辑版，用【】标录屏覆盖部分）
4. 脚本保存到 `04_脚本模板/01_待确认/`
5. 橙子说"通过#编号" → 移到 `04_脚本模板/02_待拍摄/`（同时提取元素让橙子筛选入库）
6. 橙子说"拍完了#编号" → 移到 `04_脚本模板/03_已完成/`

**路径二专用规则（主动出脚本/AI小知识系列）**：
- 主题：每天讲清一个AI小知识，目的是建立橙子"AI专家"人设
- 选题核心原则：追热度，只做当下AI圈有热度的话题（不限工具/模型，有热度就行）
- 选题流程：橙子说"写ai脚本" → 显眼包先搜当下AI热点 → 提供10个有热度的选题 → 橙子选定 → 再出脚本
- 选题标准：最近几天多平台有讨论热度 + 普通人能理解会好奇 + 不是纯技术圈自嗨
- 选题边界通过橙子/超哥的反馈动态学习，详见 `02_选题库/选题边界.md`
- ⚠️ **路径二是科普/介绍类内容，必须使用独立的脚本结构，严禁套用路径一（推文转口播）的反常识钩子+泼冷水结构**
- ⚠️ **举例必须从目标用户的真实日常场景出发**（宝妈带娃/做副业、上班族日常工作、大学生找实习找工作），**禁止用创业者/老板/HR等非目标用户的视角举例**
- ⚠️ **例子的获得感必须落到钱**：省钱/赚钱/占便宜。不能以情感、社交、成就感等软性价值收尾，要让用户感受到真实的金钱收益或节省
- ⚠️ **开头知识点名称与正文内容严格对应**：正文只讲某工具的子功能，开头就写子功能名，不能用宽泛工具名代替
- ⚠️ **例子要能撑起录屏节奏**：举例时考虑剪辑版录屏的过程感，要有多步骤/对比/有分量的结果，不能只是"输入一句话→等结果"
- 路径二专属结构：固定开头"每天分享一个AI知识，XX是什么？" → 大白话+举例子 → 收尾说明有什么用
- 脚本输出格式：一个md文件里先输出拍摄版（纯口播文字），再输出剪辑版（同样文字，用【】括出需要录屏覆盖的部分）
- 详见 `05_创作原料/06_路径二结构模板.md`

### 选题被肯定或否定时（边界学习）
1. 橙子/超哥肯定选题 → 记录到 `02_选题库/选题边界.md` 通过表，Claude总结通过原因
2. 橙子/超哥否定选题 → 记录到 `02_选题库/选题边界.md` 被否表，Claude总结被否原因
3. 每积累5条以上反馈 → 显眼包提出边界规律总结建议 → 橙子/超哥确认后再写入

### 橙子修改脚本后（学习模式）
1. 对比分析差异，提炼规律
2. 向橙子/超哥提出建议：风格文档怎么改、执行提示词怎么改、有哪些新元素值得入库
3. 橙子/超哥确认后再更新对应文件

### 通用规则
- 发现系统文件有误 → 直接更新，不用等橙子提醒
- 创作原料库只增不删（除非橙子明确说删）
- 每次生成脚本后 → 等橙子确认脚本进入"待拍摄"状态 → 显眼包提取脚本中的好元素（金句、类比、结构、节奏）→ 橙子确认哪些入库哪些丢弃
