## 谁在用这套系统

- **超哥**（老公）：负责搭建和优化工作流系统，技术侧
- **橙子**（老婆）：AI自媒体博主，负责日常使用工作流创作抖音口播脚本

**账号定位**：AI前哨，帮普通人用AI赚钱
**发布平台**：抖音
**目标用户**：宝妈、上班族、应届大学生、想做副业的普通人（AI入门级别，很多人连ChatGPT都没用过，不知道什么是提示词）

## 工作流说明

这是一套"拆、存、用"的内容生产系统：
- **拆**：橙子提供推文/爆款脚本 → Claude拆解出可复用的创作元素
- **存**：橙子筛选决定哪些入库哪些丢弃 → 好元素沉淀到元素库
- **用**：写新脚本时 → 从元素库调取钩子、案例、金句、节奏组合出稿

**日常流程**：
1. 橙子提供推文/同行爆款脚本（文字或截图）
2. Claude拆解创作元素 → 橙子筛选入库或丢弃 → 更新元素库
3. 需要出脚本时 → Claude从元素库调取元素 → 基于橙子风格生成脚本
4. 橙子修改脚本 → Claude对比分析差异 → 提炼规律更新系统文件
5. 脚本自动保存到 `04_脚本模板/历史脚本/日期_选题.md`

## 我的创作风格（核心规则）

**1. 钩子必须是反常识结论**
公式：`[大家以为正确的事] + 其实 + [反过来才对]`
钩子后面立刻跟具体工具名或案例，不绕弯子

**2. 点中用户的核心恐惧**
目标用户的真实焦虑是"被AI时代淘汰"，不是"走了弯路"

**3. 语气重、敢说**
用"屁用没有"、"大坑"、"从来不是"，不用温吞表达

**4. 画面感写法**
不写流程（"几个Agent来回开会"），写感受（"你会感觉你的算力在燃烧"）

**5. 在脚本中途主动说出用户的真实处境**
例："我估计很多人连Claude和ChatGPT都没有，都不知道啥是提示词"

**6. 用情绪词收尾，不用喊口号式行动指令**
用"焦虑内耗"、"距离你很遥远"收尾，比"现在就做一件事"更有力

**7. 具体工具名必须保留**
推文里出现的工具名（OpenClaw、ChatGPT、Claude等）不能模糊成"AI"

**8. 禁止**
- 技术词（"大模型"→"AI助手"，"prompt"→"提问方式"）
- 【】结构标注出现在脚本正文里
- 超过3个干货要点
- 喊口号式行动指令

## 每次对话的行为准则

### 收到推文/爆款脚本时（拆解模式）
1. 先判断关联度（高/中/低），低关联直接说明
2. 拆解出可复用元素，按5个维度列出：钩子模式、案例素材、金句表达、情绪节奏、结构模板
3. 等橙子筛选——橙子说"入库"的写入对应元素库文件，说"不要"的丢弃
4. 如果橙子说"先存着"，写入 `02_选题库/待用素材库.md`

### 需要出脚本时（生产模式）
1. 先读取元素库，看有哪些可复用的钩子、案例、金句
2. 基于橙子风格规则 + 元素库素材，生成脚本
3. 脚本生成后 → 立即保存为md文件到 `04_脚本模板/历史脚本/`

**路径二专用规则（主动出脚本/AI小知识系列）**：
- 主题：每天讲清一个AI小知识
- 核心圈：Claude / Claude Code / Cowork，外延圈：其他主流大模型
- 每个知识点必须落到"这跟普通人赚钱/省时间有什么关系"
- 选题边界不预设，通过橙子/超哥的反馈动态学习，详见 `02_选题库/选题边界.md`
- ⚠️ **路径二是科普/介绍类内容，必须使用独立的脚本结构，严禁套用路径一（推文转口播）的反常识钩子+泼冷水结构**
- 路径二的专属结构模板见 `05_元素库/06_路径二结构模板.md`（待橙子提供参考脚本后拆解建立）

### 选题被肯定或否定时（边界学习）
1. 橙子/超哥肯定选题 → 记录到 `02_选题库/选题边界.md` 通过表，Claude总结通过原因
2. 橙子/超哥否定选题 → 记录到 `02_选题库/选题边界.md` 被否表，Claude总结被否原因
3. 每积累5条以上反馈 → Claude主动更新"边界规律总结"，提炼什么能做、什么不能做

### 橙子修改脚本后（学习模式）
1. 对比分析差异，提炼规律
2. 更新 `01_人设系统/02_创作风格文档.md`
3. 更新 `03_工作流SOP/05_推文→脚本_执行提示词.md`
4. 如果修改中出现了新的好元素，追加到元素库

### 通用规则
- 发现系统文件有误 → 直接更新，不用等橙子提醒
- 元素库只增不删（除非橙子明确说删）

超哥和橙子在各自电脑上用Claude Code操作同一个项目，通过GitHub远程仓库同步。

| 谁说 | 说什么 | Claude做什么 |
|------|--------|-------------|
| 超哥 | **"推给老婆"** | 把所有改动commit + push到远程仓库，commit信息写清楚改了什么 |
| 橙子 | **"查看老公更新"** | 从远程仓库pull最新代码，然后用中文总结这次超哥改了什么 |
| 橙子 | **"请老公查阅"** | 把橙子的改动commit + push到远程仓库 |
| 超哥 | **"看老婆更新"** | 从远程仓库pull最新代码，总结橙子改了什么 |
